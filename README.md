# 台股策略選股系統(即時抓 + 快取、不落地版)

純 Python 的台股分析工具:開啟網頁 → 即時抓 FinMind 資料(記憶體)→ pandas 算策略 → Streamlit 顯示。**全程不寫任何檔案、不用資料庫。**

## 架構特點
- **不落地**:資料只活在記憶體,算完即丟,沒有資料庫、沒有檔案膨脹問題。
- **快取秒開**:當天第一次開會即時抓最新資料(約數十秒),之後 8 小時內再開/操作都用暫存結果秒開;過期後下次開啟才重抓。
- **資料永遠最新**:不需要排程器,更新發生在「你開啟、且快取過期」的當下。

## 技術堆疊
- 資料來源:
  - 技術面(日K/價量)→ **Yahoo 股市**(透過 yfinance,涵蓋上市/上櫃)
  - 籌碼面(三大法人)→ **臺灣證券交易所 T86**(上市)+ **櫃買中心**(上櫃),依代號自動路由
  - 基本面(月營收)→ **公開資訊觀測站 MOPS**(年增、連續成長月數、創新高)
- 分析:pandas
- 前端:Streamlit + Plotly
- 線圖支援 **日 / 週 / 月** 三種週期切換
- **強勢股排名**(3 日 / 30 日 區間漲幅)
- **依產業選股池**:可從全上市 33 個產業中挑選,動態決定追蹤範圍(非固定清單)
- 結果表與個股健檢均附 **個股 / 新聞** 連結

## 專案結構
```
.
├── config.py            # 設定:股票池、抓取區間、快取時間、Token
├── datasource.py        # 即時向 FinMind 抓資料,回傳記憶體 DataFrame(不落地)

├── dashboard/app.py     # Streamlit Dashboard(含快取)
└── requirements.txt
```

## 安裝與執行
```bash
pip install -r requirements.txt

# (選用)設定 FinMind token 提高 API 額度
#   PowerShell:  $env:FINMIND_TOKEN="你的token"   取得:https://finmindtrade.com

streamlit run dashboard/app.py     # 開啟後瀏覽 http://localhost:8501
```
不需要先灌資料,開啟網頁即會自動抓取。

## 選股條件(側欄可自由勾選 / 調參,結果即時更新)
可在網頁上切換 **AND(全部符合)/ OR(任一符合)**,並組合下列條件:

- 一鍵策略範本:⚡強勢股 / 🌱成長股(自動套用整組條件)
- 站上均線(周期可選 5 / 10 / 20 / 60 日)— 技術面,Yahoo
- 均線多頭排列(5MA>20MA>60MA)— 技術面,Yahoo
- 相對強度 RS(個股漲幅贏大盤 ^TWII)— 技術面,Yahoo
- 投信連續買超 N 天 — 籌碼面,證交所/櫃買
- 外資連續買超 N 天 — 籌碼面,證交所/櫃買
- 外資最新買超(當日) — 籌碼面,證交所/櫃買
- 自營商連續買超 N 天 / 自營商最新買超 — 籌碼面,證交所/櫃買
- 三大法人合計買超(外資+投信+自營商) — 籌碼面,證交所/櫃買
- 成交量放大(當日量 > N 日均量 × 倍數) — 技術面,Yahoo
- 突破 N 日新高 — 技術面,Yahoo
- 營收年增 ≥ X% — 基本面,MOPS
- 營收連續成長 ≥ N 月(年增連續為正) — 基本面,MOPS
- 營收創近一年新高 — 基本面,MOPS

判斷邏輯:`strategy/screener.py` 的 `build_metrics` 先算齊各檔指標,
`apply_filters` 再依勾選的條件以 AND/OR 組合過濾,是可擴充的規則引擎。
健檢評分(`score_stock`)同時涵蓋 **技術 + 籌碼 + 基本面** 三個維度。

## 個股健檢(輸入任意代號)
網頁下方可輸入任一股票代號(支援上市/上櫃),即時抓取後:
- 顯示收盤、漲跌、各項技術/籌碼指標與日/週/月線圖
- 給出 **0~100 健檢評分** 與綜合研判,並附逐項加分明細(`score_stock`)
- 附上 Yahoo 股市 / Goodinfo / 玩股網 / 公開資訊觀測站 連結

> ⚠️ 健檢評分是把多空訊號用固定權重量化的**機械式參考分數,非投資建議**,
> 不保證漲跌。上櫃股目前無證交所三大法人資料,籌碼面項目以 0 計分。

> 註:月營收(基本面)取自 MOPS,月更新、快取 24 小時;`REVENUE_MONTHS`
> 控制回抓月數(預設 13,故「連續成長月數」上限為該窗口)。金融股營收年增
> 參考意義較低。獲利面(EPS / 毛利率)需季報,尚未納入,可後續再補。

## 限制與後續
- **股票池不宜過大**:每次抓取是即時連線,幾十檔內體驗良好;要追蹤全市場需改回「有資料庫」的架構(避免每次重抓上千檔)。
- **無歷史紀錄**:不存資料,故無法回測「過去某天選了哪些股、後來漲跌」。要回測時,再加「只存精簡選股結果」即可。
- 金融股(如富邦金)的月營收年增率不具參考意義,建議篩選時排除或改用其他基本面指標。
